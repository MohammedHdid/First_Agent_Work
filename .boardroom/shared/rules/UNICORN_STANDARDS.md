# THE UNICORN STANDARDS

> Non-negotiable quality gates for any output generated by any agent in either system.
> Applies to: Antigravity system + Claude Code system

---

## 1. Performance
- **Lighthouse Score**: Must be > 95 across all metrics (Performance, Accessibility, Best Practices, SEO).
- **Asset Optimization**: All images must be WebP/AVIF. Lazy loading is mandatory. Use SSR or SSG via Next.js where applicable. For Express (HTML) mode: minify CSS/JS, compress images before embedding.

## 2. Accessibility
- **Standard**: WCAG 2.1 AA Compliance.
- **Requirements**: Proper ARIA labels, semantic HTML tags, keyboard navigability, high contrast ratios (4.5:1 minimum for body text).

## 3. Design Aesthetics
- **Core Theme**: "Unicorn" — Ferrari performance meets Tractor utility. Premium look, fast delivery.
- **Visuals**: Glassmorphism where appropriate (translucency and background blur on cards/modals).
- **Lighting**: Light-leaks and subtle gradient meshes on dark backgrounds.
- **Motion**: Smooth transitions for page entrances and micro-interactions. Elements should feel responsive and "alive". Use CSS transitions for Express mode, Framer Motion for Next.js projects.
- **Typography**: Modern, premium sans-serif (Inter, Outfit, or as specified) with clear typographic hierarchy. H1 > H2 > H3 > body — never invert.

## 4. Stack-Specific Standards

### Express Mode (HTML/CSS/JS)
- Vanilla CSS with custom properties (`--brand-primary`, etc.)
- CSS transitions instead of Framer Motion
- Images: compressed WebP
- Deploy: Vercel static (zero config)

### Freestyle / Boardroom Mode (Next.js)
- Tailwind CSS 4 + Framer Motion
- Shadcn/UI components (customized)
- next/image for all images
- Google Fonts: Inter (body) + Outfit (headings) unless CONTRACT specifies otherwise

## 5. Mobile-First
- Every layout must be designed mobile-first: 375px → 768px → 1280px → 1440px
- No horizontal overflow at any breakpoint
- Touch targets minimum 44×44px
